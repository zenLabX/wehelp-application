1 個人簡介和申請動機

哈囉！我是黃鈺婷，興趣是看攝影展、學習跟攝影史有關的一切人事物。中央英文系畢業後，在編輯相關領域工作 5 年，後來在六角學院自學 9 個月後就轉職到軟體業，目前一共在駐點及專案團隊累積 3 年的軟體工程師經驗。

這 3 年經過探索和嘗試，發現我的熱忱在於打造注重使用者體驗的產品或是 SaaS 服務，這是我在專案公司無法獲得的經驗。我喜歡探究一個系統各項功能產生的背後原因，而不只僅僅是完成一個又一個的專案，卻都不明白功能之間的關聯性和對使用者真實的幫助？然而，在轉職期間發現自己的基本功不扎實、自學速度慢，讀文件能力一直沒有進步，自行研究技術的速度及深度還有待加強，因此一直不太有相關公司的面試機會，或即便有機會，也因為能力問題沒有被錄取。

今年在經歷幾場面試後，發現 CI/CD、寫測試、容器化部署是產品及 SaaS 服務公司的基本條件，過去專案開發都是在內網環境，也想透過這次課程接觸 AWS 雲端服務的使用。此外，我發現自己對於研究「如何學習」很感興趣，所以已經有初步規劃想製作的專題。

目前在 9 月初已辭去工作待業中，想專心準備轉往產品類型公司，希望透過加入 WeHelp 訓練營，打造自己也會很愛用的軟體，與志同道合的同學們一起往目標前進！

5 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

近期曾與主管理念不合，當我接到一個專案環境部署工作，他認為我應該直接動手開始編譯程式、部署環境，而我則是在準備檔案的過程發現跟他前天說的需求不太一樣，想要先做確認。後來他劈頭講了非常多對我的看法，以及拿我跟其他同事比較。我委婉的回覆他，「每個人是不同的，沒辦法這樣比較」，但他說：「公司就是這樣在比較！」

我認為遇到負面情緒時，需要重新檢視外界的衝擊事件，是不是真的有我能調整的？產生不舒服時，通常都被我視為可能是改變的契機。如果有，那就去調整，都跌倒了，至少抓把沙子再站起來吧！如果自己已經盡力了，內耗也沒有用，就繼續往自己目標前進就好了。同時，也會尋求有相關經驗的朋友或前輩詢問，有時候也會發現是自己經驗歷練不夠，而有一些盲點看不到才導致負面情緒，通常在看懂後就會豁然開朗！

6 關於這份申請網頁，分享一個開發時的技術心得

在 commit 程式碼過程，為了想另開一個新的 GitHub 帳號，本地電腦遇到兩個 GitHub 帳號在 commit 時的權限打架的問題。由於之前只有一個帳號時，都是用 Https 驗證，現在同時擁有兩個帳號，當要把程式碼從本地端 commit 到其中一個帳號時，就必須在該帳號的 GitHub 上去產生 token 才能有個別 repository 的 commit 權限。因為接下來會以新帳號為主，所以後來我以新帳號重新申請 SSH Key 讓身份驗證一勞永逸，跟著 AI 指示一步步產生本地端 private key，並且把本地端產的 public key 放到 GitHub 上，也藉此更了解 private key/public key 的使用。

7 如何看待人工智慧的發展對軟體工程師的影響？

  1. 業界還是有很多內部老舊的大型系統，這些系統的維護擴充需求應該還會持續下去，這類的工程師職務需求應該還會存在個至少 5-10 年。
  2. 未來工程師要有能夠將人類的需求轉化成程式碼的能力，可能不再是坐等 SA、SD、設計師給一個清楚需求才能開始開發，而是要能和團隊協作溝通，我認為好的產品都是透過集思廣益討論出來的，應該很難是一個人的決定(除非他是天才或有讀心術的特異功能)。其實也就是釐清需求的能力會是最重要的，知道該問什麼問題引導需求被釐清，因為客戶大部分不知道自己要什麼...。
  3. 未來的工程師不止是寫出程式，還需要寫出好擴充、好維護的程式碼，因為 AI 時代下，需要因應更多更快的需求變動應該也是有可能的。
  4. 會讀程式碼比會寫程式更重要，接手不同專案時，只要能看懂前人的商業邏輯，轉化成需求請 AI 幫忙，都可以加快工作效率了。
  5. AI 產出大量的程式碼 code base，在沒有 code review 的團隊，未來可能產生資安問題，我認為資安需求會因此增加，培養資安方面的知識可能變得更重要。
